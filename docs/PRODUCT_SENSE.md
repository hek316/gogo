# PRODUCT_SENSE.md — GoGo 제품 감각

이 문서는 GoGo가 **누구를 위해, 어떤 문제를 해결하는지** 정의합니다.
새 기능을 추가할지 말지 판단이 서지 않을 때, 먼저 여기를 읽으세요.

---

## 1. 우리가 해결하는 문제

### 실제 장면
> 친구들과 카톡방에서 "어디 갈까?" 할 때,
> 누군가가 예전에 저장해 둔 맛집 링크를 찾느라 스크롤을 한참 올린다.
> 결국 기억이 안 나서 그냥 아무 데나 간다.

GoGo는 이 장면을 없애기 위해 만들어졌어요.

**핵심 문제 세 가지:**
1. **흩어진 장소 기록** — 맛집은 카카오맵 저장, 인스타그램 북마크, 카톡 대화에 뒤섞여 있다
2. **공유의 마찰** — 내가 저장한 장소를 친구들에게 보여주려면 링크를 일일이 보내야 한다
3. **약속 잡기의 비효율** — "어디 갈지"를 결정하는 데 대화가 너무 많이 필요하다

---

## 2. 타겟 유저

### 주요 타겟: 2-6명의 소규모 친구 그룹
- 자주 함께 외식하거나 여행하는 사이
- 카카오톡으로 약속을 잡는 한국 사용자
- 앱 설치나 회원가입에 거부감이 있는 사람들 포함

### 유저 페르소나
**기록하는 사람 (주도자)**
- 맛집을 자주 저장하고, 친구들에게 공유하고 싶어 한다
- 지금은 카카오맵 즐겨찾기나 인스타 저장에 흩어져 있어서 불편하다
- "우리 다음엔 여기 가자"를 말하고 싶을 때 바로 꺼낼 수 있는 곳이 필요하다

**따라가는 사람 (참여자)**
- 주도자가 링크 보내주면 참여하는 타입
- 앱 설치나 가입하기 귀찮다
- 투표하고, 결과 보고, 후기 보는 정도면 충분하다

## 3. 핵심 사용자 여정

```
[장소 발견]
  맛집 인스타 or 네이버 블로그 링크 발견
        ↓
[GoGo에 등록]
  URL 붙여넣기 → 자동으로 이름/사진 채워짐
        ↓
[그룹에 공유]
  친구들 카톡방에 초대 링크 하나 → 가입 없이 참여
        ↓
[같이 고르기]
  미팅 생성 → 후보 장소 투표
        ↓
[방문 후]
  후기 남기기 → WANT_TO_GO → VISITED 상태 변경
```

이 흐름이 **5분 안에** 끝나야 합니다. 어떤 단계에서도 "이거 어떻게 하지?"라는 생각이 드면 안 돼요.

---

## 4. 우리가 하지 않는 것 (Anti-goals)

기능 추가를 요청받을 때 아래 기준으로 거절할 수 있어요.

### ❌ 소셜 피드 / 팔로워 시스템
GoGo는 인스타그램이 아닙니다. 불특정 다수에게 내 장소를 보여주는 것은 우리 문제가 아니에요. 친한 사람들끼리의 공유가 핵심입니다.

### ❌ 정밀한 권한 관리
"그룹 관리자만 장소를 추가할 수 있다"같은 세밀한 권한 설정은 오버엔지니어링입니다. 친구 사이에 권한 체계는 어색해요.

### ❌ 알림/푸시 시스템 (현재)
실시간 알림은 인프라 복잡도를 크게 높입니다. 지금은 "들어가서 확인하는" 방식으로도 충분합니다.

### ❌ 장소 리뷰의 플랫폼화
별점 집계, 랭킹, 인기 장소 노출 같은 기능은 네이버 지도나 카카오맵이 더 잘합니다. 우리는 "우리끼리의 기록"에 집중합니다.

---

## 5. 제품이 성공했다는 신호

### 단기 지표 (지금 관심 가질 것들)
- 그룹 생성 후 **초대 링크를 실제로 사용한 비율** (링크만 만들고 혼자 쓰면 의미 없음)
- 장소 등록 후 **미팅으로 이어진 비율** (기록만 하고 끝나면 반쪽짜리)
- 미팅 투표 참여 인원 (보통 몇 명이 참여하는지)

### 장기 지표 (나중에 볼 것들)
- 그룹당 평균 재방문 횟수 (한 번 쓰고 버리는 게 아닌지)
- VISITED 전환율 (실제로 방문했는지)

---

## 6. 기능 추가 판단 기준

새 기능 아이디어가 나왔을 때 아래 질문을 순서대로 해보세요.

1. **"주도자의 삶을 더 쉽게 만드는가?"** → 그렇다면 강력한 후보
2. **"참여자의 마찰을 줄이는가?"** → 그렇다면 고려할 만함
3. **"가입 없이 할 수 있어야 하는가?"** → 핵심 기능이라면 그래야 함
4. **"이게 없어도 핵심 여정이 동작하는가?"** → 없어도 된다면 나중에

> 기능이 많을수록 좋은 앱이 아닙니다.
> **덜 고민하게 만드는 앱**이 좋은 앱입니다.

---

## 7. 현재 제품의 경계와 다음 단계

### 지금 할 수 있는 것
- 장소를 URL로 등록하고 목록으로 관리
- 그룹을 만들고 초대 링크로 친구 초대 (닉네임만으로 참여 가능)
- 그룹 내에서 장소 공유 및 미팅 투표
- 방문 후 후기 작성, 좋아요

### 명확히 없는 것 (요청받아도 신중해야 할 것)
- 실시간 알림
- 장소 지도 시각화
- 공개 장소 탐색 피드
- 날짜/시간 기반 약속 잡기 (현재는 장소 투표만 존재)

### 자연스러운 다음 단계 (언젠가)
- 닉네임 ↔ 로그인 계정 연결 (지금은 완전히 분리됨)
- 방문 일정 잡기 (장소 결정 후 날짜까지)
- 장소 카테고리/태그 기반 필터링 강화
