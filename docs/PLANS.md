# PLANS.md — GoGo 로드맵

이 문서는 GoGo의 현재 상태와 앞으로의 방향을 정리합니다.
세부 실행 계획은 `docs/exec-plans/active/`에, 기술 부채는 `docs/exec-plans/tech-debt-tracker.md`에서 관리합니다.

**마지막 업데이트**: 2026-04-04

---

## 1. 현재 상태 (What's Done)

### 완성된 기능

**장소 관리**
- 장소 등록 (URL 자동 프리뷰 — 제목/이미지/주소 스크래핑)
- Naver Local API 기반 장소 검색
- 장소 목록 조회 (전체 / 카테고리 필터)
- 장소 상세 조회
- 장소 삭제
- 방문 처리 (`WANT_TO_GO` → `VISITED`)
- 좋아요 / 좋아요 취소
- 인기 장소 / 최근 등록 장소 조회

**그룹**
- 그룹 생성
- 초대 코드(8자)로 그룹 참여 (로그인 불필요)
- 그룹 내 장소 공유
- 그룹 상세 조회

**약속 (Meeting)**
- 미팅 생성 (후보 장소 선택)
- 장소 투표 / 재투표
- 투표 결과 확인
- 약속 장소 확정

**후기**
- 방문 후기 작성 (별점 + 텍스트)
- 장소별 후기 목록 조회

**인증**
- 카카오 OAuth 로그인
- Google OAuth 로그인
- JWT 액세스 토큰 (15분) + 리프레시 토큰 로테이션 (7일)
- 로그아웃

**프론트엔드 페이지**
- 홈 (`/`) — 전체 장소 목록
- 장소 목록 (`/places`) — 내 장소 필터
- 장소 상세 (`/places/[id]`)
- 탐색 (`/explore`) — 인기/최근 장소
- 그룹 목록 (`/groups`)
- 그룹 상세 (`/groups/[id]`)
- 미팅 상세 (`/meetings/[id]`)
- 프로필 (`/profile`)
- 로그인 (`/auth/login`)

## 3. 다음 단계 (Next)

품질 안정화가 끝난 뒤 집중할 기능/개선 영역입니다.

### 3-1. 기능 개선

**약속 플로우 완성**
현재 약속은 "장소 투표 + 확정"만 있습니다. 실제 약속으로 이어지려면 날짜/시간 선택이 필요합니다.
- 미팅에 날짜 후보 추가 기능
- 날짜 + 장소 동시 투표 또는 순차 투표

**그룹 경험 개선**
- 그룹 초대 링크 공유 버튼 (카카오 공유 등)
- 그룹 내 최근 활동 표시 (어떤 장소가 추가됐는지)

**장소 경험 개선**
- 장소 수정 기능 (현재 삭제만 가능)
- 이미지 직접 업로드 (현재 URL 스크래핑만)

### 3-2. 기술 부채 해소

우선순위 순:
1. `REL-005` — Flyway 도입 (스키마 마이그레이션 이력 관리)
2. `ARCH-002` — API 에러 코드 체계 도입
3. `ARCH-003/004` — `FetchType.EAGER` → LAZY + fetch join 전환
4. `REL-004` — Sentry 에러 트래킹 연동

### 3-3. 개발 경험

- CI/CD 파이프라인 구성 (GitHub Actions → ECR push → EC2 배포 자동화)
- PR 머지 시 테스트 자동 실행 + 커버리지 게이트

## 4. 미래 방향 (Later)

지금 당장은 아니지만, 사용자 수가 늘어날 때 고려해야 할 것들입니다.

### 4-1. 인프라 스케일업

현재 EC2 단일 인스턴스 + 로컬 PostgreSQL 구조는 SPOF입니다.

```
현재:
  EC2 단일 인스턴스
    └── Docker: backend + postgres

미래:
  ALB
    ├── EC2 (AZ-a)
    └── EC2 (AZ-b)
  RDS Multi-AZ PostgreSQL
```

**트리거**: 동시 사용자 수가 늘어 응답 지연이 체감될 때, 또는 첫 번째 DB 관련 장애가 발생했을 때.

### 4-2. 닉네임 → 계정 기반 전환

현재 그룹 멤버, 투표가 닉네임(String)으로만 연결됩니다. 계정이 연결되면:
- 내가 참여한 그룹 전체 조회 가능
- 투표 중복/위장 방지
- 방문 이력 계정에 귀속

`tech-debt-tracker.md` `ARCH-001` 참조. 마이그레이션 난이도가 높으므로 신중하게.

### 4-3. 알림

지금은 앱에 들어와야 새 투표, 약속 확정을 알 수 있습니다.
- 웹 푸시 알림 (PWA)
- 또는 카카오 알림톡 (약속 확정 시)

**트리거**: 사용자들이 "알림이 없어서 불편하다"는 피드백이 반복될 때.

### 4-4. 지도 시각화

장소 목록을 지도에서 볼 수 있는 기능.
- 카카오맵 또는 Naver Maps API 연동
- 그룹 내 공유 장소를 지도 핀으로 표시

`PRODUCT_SENSE.md`의 Anti-goals와 충돌하지 않는 범위에서. "지도 앱이 아닌 것"을 유지하면서 보조 뷰로만 제공.

## 5. 하지 않을 것

아무리 요청이 와도 GoGo의 방향에 맞지 않는 것들입니다.
상세 이유는 `docs/design-docs/core-beliefs.md`와 `docs/PRODUCT_SENSE.md` 참조.

| 기능 | 이유 |
|------|------|
| 소셜 피드 / 팔로워 | 우리는 "친한 친구끼리"에 집중 |
| AI 장소 추천 | 결정을 돕는 도구, 결정을 대신하지 않음 |
| 공개 오픈 커뮤니티 그룹 | 소규모 친밀한 그룹이 핵심 |
| 인앱 채팅 | 카톡이 이미 있음 |
| 복잡한 권한 체계 | 친구 사이에 역할 구분은 어색 |
